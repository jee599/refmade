Broken Password Reset Link

A spike in users clicking on reset links but not completing the reset flow were detected. Error rates surged ten minutes after the latest deployment.

Critical Open In Progress
Root Cause Analysis

BugShield flagged this incident after observing a surge in failed password reset attempts.

Detection showed a spike in users clicking on email links but not completing the reset flow. Token validation endpoint returning 403 Forbidden for all requests.

Activity
SK
Sarah Kim · 45 min ago

Research says this was caused by a stale JWT signing key after the deployment rotated secrets, but the cache never invalidated the old key.

MR
Mike Rivera · 20 min ago

Patch submitted. Token cache now flushes on deployment. Reset tests were closed successfully, but users never reached the final "Password Updated" confirmation screen.
